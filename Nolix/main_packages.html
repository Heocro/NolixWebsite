Main packages



ch.nolix.primitive Contains base classes for tests and classes with validation functions. These very basic functionalities have no dependencies. Therefore they do not import potential bugs from other classes.
ch.nolix.core Contains classes for general purpose.
ch.nolix.element Contains element classes. Elements are reproducable. That means that elements can be represented by a string or specification file.
ch.nolix.system Contains classes for general purpose which uses elements.
